# ChiyanMap 前端渲染验证系统 — 技术文档

## 概述

本系统构建于 Minecraft Bedrock 模组 **ChiyanMap** 之上，利用 **DirectX 11 Present Hook + Dear ImGui** 技术栈，在游戏窗口内部实现了一套完整的前端渲染能力验证框架。其核心目标是以可视化的方式验证 ImGui 在游戏叠加层中的渲染边界与表现能力。

## 一、系统架构

### 1.1 整体渲染流水线

```
Minecraft Bedrock Client (D3D11)
    │
    ▼
IDXGISwapChain::Present (vtable[8])
    │
    ├─ MinHook 拦截 → DX11Hook::hkPresent()
    │       │
    │       ▼
    │   DX11Hook::RenderImGui()
    │       │
    │       ├─ ImGui_ImplDX11_NewFrame()
    │       ├─ ImGui_ImplWin32_NewFrame()
    │       ├─ ImGui::NewFrame()
    │       │
    │       ├─ VerificationUI::RenderVerificationFrame()    ← 验证系统入口
    │       │       │
    │       │       ├─ HandleNKeyPress()                    ← N 键状态机
    │       │       ├─ RenderMainConsole()                  ← 验证主菜单
    │       │       ├─ RenderThemeShowcase()                ← 模块1 主题换肤
    │       │       ├─ RenderHUDOverlay()                   ← 模块2 HUD叠加
    │       │       ├─ RenderControlGallery()               ← 模块3 控件全展示
    │       │       ├─ RenderDockSpace()                    ← 模块4 DockSpace
    │       │       ├─ RenderStressTest()                   ← 模块5 压力测试
    │       │       ├─ RenderDataTable()                    ← 模块6 大数据表格
    │       │       └─ RenderComplexGUI()                   ← 模块7 复杂风格GUI
    │       │
    │       ├─ RenderImGuiBigMap()          ← 原有大地图
    │       ├─ RenderImGuiWaypointUI()      ← 原有路径点管理
    │       │
    │       ├─ ImGui::Render()
    │       └─ ImGui_ImplDX11_RenderDrawData()
    │
    ▼
游戏原始 Present 调用继续
```

## 二、按键系统

| 按键 | 功能 | 实现位置 |
|------|------|---------|
| **N** | 三态切换：关闭 ↔ 主菜单 ↔ 子模块 ↔ 回到主菜单 | `VerificationUI::HandleNKeyPress()` |
| **M** | 大地图（原有功能保留） | `DX11Hook.h` 内原有 |
| **U** | 路径点管理（原有功能保留） | `DX11Hook.h` 内原有 |
| **F11** | 游戏全屏切换（透传） | `hkGetAsyncKeyState` 中排除 |
| **Esc** | 关闭大地图/路径点菜单 | `WndProcHook` 中拦截 |

N 键的三态切换逻辑：

```
状态: [无UI]
  │ N 键
  ▼
状态: [主菜单] showVerificationConsole=true, activeModule=0
  │ 点击卡片
  ▼
状态: [子模块] activeModule=1~7
  │ N 键 (回到主菜单)
  ▼
状态: [主菜单] activeModule=0
  │ N 键 (关闭全部)
  ▼
状态: [无UI]
```

    │   ├─ WindowBorderSize / FrameBorderSize / TabBorderSize
    │   └─ ScrollbarSize / GrabMinSize
    │
    └─ 填充 50+ ImGuiCol_* 颜色槽
        ├─ ImGuiCol_Text / TextDisabled
        ├─ ImGuiCol_WindowBg / ChildBg / PopupBg
        ├─ ImGuiCol_Border / BorderShadow
        ├─ ImGuiCol_FrameBg / FrameBgHovered / FrameBgActive
        ├─ ImGuiCol_TitleBg / TitleBgActive
        ├─ ImGuiCol_Button / ButtonHovered / ButtonActive

        ├─ ImGuiCol_Table 系列

## 五、复杂风格 GUI 详解（模块7）

第7模块是整个验证系统中最复杂的部分，包含 6 个子标签页，每个子标签页覆盖一种高级 GUI 风格。

### 5.1 动画效果 (Animation)

| 验证项 | 技术实现 | 核心代码 |
|--------|---------|---------|
| 呼吸脉冲按钮 | `sinf(t*3)` 计算颜色权重，PushStyleColor 实时覆盖 | `pulse = (sinf(t*3)+1)*0.5` |
| 旋转加载动画 | 12个圆形等角分布，`cosf/sinf` 计算位置，alpha 渐变 | `angle = t*3 + i*π/6` |
| 进度条动画 | `fmodf(t*0.2, 1.0)` 循环进度，颜色渐变 | `progCol HSV 色相偏移` |
| 滑入按钮 | 三角波函数 `2-2*abs(t*2-1)` 控制缩进和透明度 | `PushStyleVar(Alpha, slideOff)` |
| 流光文字 | `ImColor::HSV(hue, 0.8, 1.0)` 每帧更新 hue | `hue = fmodf(t*0.3, 1.0)` |

### 5.2 动态颜色 (Dynamic Color)

| 验证项 | 技术实现 |
|--------|---------|
| HSV 色相环 | `ImGui::ColorPicker4()` + `ImGuiColorEditFlags_PickerHueWheel` |
| 渐变背景条 | `ImColor::HSV(pct + t*0.05, 0.8, 1.0)` 逐像素绘制，每帧滚动 |
| 呼吸标签 | 8个按钮各自维持相位 `t*1.5 + i*0.8`，HSV 循环 |
| 动态温度计 | `(sinf(t*0.8)+1)*0.5` 控制值，颜色从红到绿渐变 |

### 5.3 MC 原生风格 (Minecraft Style)

| 验证项 | 实现方法 |
|--------|---------|
| 背包面板 | `PushStyleColor(ChildBg)` 暗色石板背景 + `ChildRounding=2` |
| 4x3 物品网格 | 12个按钮模拟物品格，`cellW` 等宽计算 |
| 热键栏 (1-9) | 9个紧凑按钮，选中项加亮 |
| MC 操作按钮 | 绿色(创建)/红色(删除)/蓝色(设置)，`FrameRounding=1` 像素风 |

### 5.4 毛玻璃效果 (Glassmorphism)

毛玻璃效果通过多层叠加绘制实现：

```
背景层 (彩色格子)
    │
    ▼
毛玻璃面板层:
    1. AddRectFilled(Alpha=30)    → 半透明底色
    2. AddRectFilled(Alpha=15)    → 内发光层
    3. AddRect(Alpha=80, 1.5px)   → 边框光晕
    4. AddRect(Alpha=40, 1px)     → 外发光边缘
    │
    ▼
文字内容 (Alpha=220)
```

### 5.5 Linear 极简 (Linear Minimal)

通过临时覆盖 `ImGuiStyle` 的颜色数组和样式变量实现：

```cpp
// 临时保存原风格
ImVec4 origColors[ImGuiCol_COUNT];
memcpy(origColors, origStyle.Colors, sizeof(origColors));

// 设置 Linear 风格
for (int i = 0; i < ImGuiCol_COUNT; i++) {
    if (i == WindowBg || i == ChildBg)  → 全透明
    if (i == Button || i == FrameBg)    → 全透明
    if (i == ButtonHovered)              → 极淡蓝 12%
    if (i == Border)                     → 半透明蓝 50%
}
// ... 展示控件 ...
// 恢复原风格
memcpy(origStyle.Colors, origColors, sizeof(origColors));
```

### 5.6 代码编辑器 (Code Editor)

语法高亮通过手动逐词分词 + `ImDrawList` 分段着色实现：

```
每行文本
    │
    ├─ getSegmentColor() 快速检测:
    │   ├─ "//" 开头 → 注释色 (绿灰)
    │   ├─ "#"  开头 → 预处理器色 (蓝)
    │   └─ 其他 → 进入逐词分词
    │
    └─ 逐词分词循环:
        ├─ isWordChar → 积累 word[]
        ├─ flush 时匹配:
        │   ├─ class/public/if/return → 关键字色(蓝)
        │   ├─ Renderer → 类型色(青)
        │   ├─ getInstance → 函数色(黄)
        │   └─ m_initialized → 变量色(橙)
        ├─ '"' → 字符串色(绿), 跳至闭合引号
        ├─ 标点 → 标点色(灰)
        └─ 空格 → 推进光标
```

**语法高亮颜色映射**：

| Token 类型 | 颜色 | RGB |
|-----------|------|-----|
| 关键字 | 蓝色 | `(120, 150, 255)` |
| 类型名 | 青色 | `(120, 200, 220)` |
| 函数名 | 黄色 | `(200, 180, 100)` |
| 字符串 | 绿色 | `(150, 230, 150)` |
| 数字/变量 | 橙色 | `(255, 200, 120)` |
| 注释 | 灰绿 | `(100, 120, 100)` |
| 标点 | 灰 | `(180, 180, 200)` |
| 普通文字 | 浅灰 | `(200, 200, 220)` |
| 行号 | 暗灰 | `(80, 90, 110)` |
| 当前行背景 | 暗蓝 | `(15, 20, 30)` |

## 六、输入拦截系统

当验证系统 UI 激活时（`showVerificationConsole` 或 `activeVerificationModule != 0`），输入拦截系统会阻止游戏响应玩家操作：

### 6.1 拦截层次

```
玩家操作 (键盘/鼠标)
    │
    ├── WndProcHook (窗口过程)
    │   ├─ ESC → 关闭大地图/路径点
    │   ├─ WM_INPUT → 阻断
    │   ├─ WM_MOUSEFIRST~LAST → 阻断
    │   ├─ WM_KEYFIRST~KEYLAST → 阻断 (除F11)
    │   └─ WM_SETCURSOR → 强制IDC_ARROW
    │
    ├── GetRawInputData Hook → 返回-1
    ├── GetRawInputBuffer Hook → 返回-1
    ├── GetAsyncKeyState Hook → 返回0 (除 N/M/F11)
    ├── GetKeyState Hook → 返回0 (除 M/F11)
    │
    ├── PlayerHook (游戏行为):
    │   ├─ _applyTurnDelta → 禁止转向
    │   ├─ startDestroyBlock → 禁止破坏
    │   ├─ attack → 禁止攻击
    │   ├─ useItem → 禁止使用物品
    │   ├─ swing → 禁止挥动
    │   ├─ _applyInputLocks → 禁止移动
    │   └─ ... (共18个 Hook)
    │
    ▼
游戏完全锁死
```

### 6.2 排除白名单

| 按键 | 用途 | 允许条件 |
|------|------|---------|
| N (0x4E) | 切换验证系统 | 始终允许 (GetAsyncKeyState) |
| M (0x4D) | 大地图 | 始终允许 (GetAsyncKeyState + GetKeyState) |
| F11 | 全屏切换 | 始终允许 (两者均允许) |

| 大数量表格性能 | 10000+ 行会明显掉帧 |

### 8.3 核心发现

1. **ImGui 样式系统完全支持多主题**：通过 `GetStyle().Colors[i]` 覆盖 50+ 颜色槽 + 样式变量，可实现与网页 CSS 类似的多主题切换能力
2. **动画系统依赖每帧计算**：ImGui 没有内置动画引擎，所有动画需通过 `ImGui::GetTime()` 每帧计算颜色/位置/透明度
3. **批量绘制性能瓶颈**：ImDrawList 在大批量（>10000 图元）下有明显 FPS 下降，建议游戏内 HUD 控制在 2000 图元以内
4. **语法高亮可行但需手动实现**：ImGui 没有富文本控件，语法高亮需通过逐词分词 + AddText 分段绘制
