> 版本：v1.0

## 一、概述

FR 设置菜单是 FastRenderer 框架的核心管理界面，提供对所有功能模块的统一配置入口。用户可通过导航快速切换功能页，完成资源管理、按键绑定、系统设置等操作。

### 设计目标

- **窗口样式**：左右布局，导航栏 + 内容区
- **导航页面**：仪表盘、资源管理、按键绑定、设置
- **按键绑定**：支持更改、冲突检测与提示
- **生产可用**：功能完善、交互完整、适合真实业务

### 3.3 按键绑定页面

核心功能页。支持查看、修改按键绑定，检测并解决冲突。

```
┌────────────────────────────────────────────────────┐
│ ⌨️ 按键绑定                                         │
├────────────────────────────────────────────────────┤
│ 插件: [FRTest-Native ▼]   ← 按插件筛选              │
│                                                    │
│ ┌────┬────────────┬──────────┬──────────┬──────┐   │
│ │ 键 │ 绑定 ID    │ 名称     │ 插件     │ 操作 │   │
│ ├────┼────────────┼──────────┼──────────┼──────┤   │
│ │ K  │ dashboard  │ 仪表盘   │ FRTest   │ [修改]│   │
│ │ N  │ sandbox    │ 沙盒     │ FRTest   │ [修改]│   │
│ │ F3 │ f3_debug   │ F3调试   │ FRTest   │ [修改]│   │
│ │ F6 │ menu       │ 全屏菜单 │ FRTest   │ [修改]│   │
│ ├────┼────────────┼──────────┼──────────┼──────┤   │
│ │    │ [未绑定]   │          │          │ [+]  │   │
│ └────┴────────────┴──────────┴──────────┴──────┘   │
│                                                    │
│ ⚠️ 冲突检测:                                        │
│ 键 `N` 已被绑定到 "切换控件沙盒" 和 系统控制台       │
│ [忽略冲突]  [更改按键 → __]                         │
│                                                    │
│ [恢复默认]  [应用更改]                               │
└────────────────────────────────────────────────────┘
```

**功能清单：**
- 表格展示所有按键绑定（键位、ID、名称、插件）
- 按插件筛选
- 点击「修改」进入按键捕获模式（按下新键绑定）
- 自动检测冲突键（同一键位被多个绑定使用）
- 冲突时显示冲突列表 + 解决建议
- 支持「忽略冲突」或「更改按键」
- 恢复默认绑定、应用更改

**冲突检测逻辑：**

```
用户按 F3 绑定 "toggle_menu"
  → 检查 g_keybinds 中是否已存在 F3
  → 若存在：显示 "键 F3 已被 'toggle_f3_debug' (FRTest-Native) 占用"
  → 选项 A：忽略冲突（覆盖原绑定）
  → 选项 B：取消操作
  → 选项 C：自动为原绑定分配新键
```

**按键捕获流程：**

```
1. 用户点击 [修改] → 进入捕获模式
2. 按钮变为 "按新键..." 闪烁
3. 用户按下目标键
4. 系统捕获按键 → 检测冲突
5. 无冲突 → 直接绑定
6. 有冲突 → 弹出冲突提示
7. 用户选择处理方式
```

## 六、技术要点

### 6.1 页面切换

通过 FRTest-Native 的状态机实现：

```
用户点击导航 → gui_event("settings_menu", "nav_dashboard")
  → FRTest-Native 收到事件
  → 更新 g_currentPage = "dashboard"
  → 重新注册 settings_menu GUI
  → 客户端收到新 GUI 定义 → 渲染新的页面内容
```

### 6.2 按键捕获

```
用户点击 [修改] → gui_event("settings_menu", "kb_capture_start|dashboard")
  → FRTest-Native 设置 g_capturingBindId = "dashboard"
  → GUI 上显示 "按新键..."
  → 用户按下 K 键
  → keybind_event("keydown") → FRTest-Native 接收
  → 检测冲突
  → GUI 显示冲突结果
```

### 6.3 冲突解决策略

| 策略 | 行为 |
|------|------|
| **覆盖** | 新绑定占用该键，原绑定被取消 |
| **交换** | 两个绑定互换键位 |
| **取消** | 不做任何更改 |
