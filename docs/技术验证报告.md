### 下一步：极限验证内容

#### 1. 性能极限
- **目标**: 找到 ImGui 在单帧内能渲染的顶点/ DrawCall 上限
- **测试**: 用 ImDrawList 批量绘制 1000 / 10000 / 100000 个图元，记录 FPS 降幅

#### 2. 渲染极限
- **目标**: 验证自定义纹理、RenderTarget、多重采样
- **测试**: 加载外部 PNG/JPG 贴图 → 创建 D3D11 ShaderResourceView → 在 ImGui 中 Image 显示
- **测试**: 渲染到纹理 → 在 ImGui 窗口显示渲染结果（延迟渲染 POC）

#### 3. 交互极限
- **目标**: 验证丰富的输入/交互模式
- **测试**: 拖拽重排（ImGui drag and drop）、自定义绘制（ImDrawList 曲线/贝塞尔/扇形）、多窗口同步、可停靠窗口（DockSpace）

#### 4. 数据展示极限
- **目标**: 验证大规模数据展示
- **测试**: ImGui Table 渲染 1000 行数据、虚拟滚动、动态热力图、折线图/柱状图（ImPlot）

#### 5. 3D - 2D 混合渲染
- **目标**: 验证游戏世界空间 → 屏幕空间的投影
- **测试**: D3D11 场景渲染到纹理 → ImGui Image 展示 → 路径点/实体标记覆盖

### 实施顺序

```
Step 1: 自定义纹理加载 (PNG/DDS → SRV → ImGui::Image)
Step 2: 批量图元压力测试 (100~100K draw calls)
Step 3: ImGui Table + 大列表虚拟滚动
Step 4: ImPlot 图表集成 (折线图/直方图)
Step 5: Render-to-Texture → ImGui 展示 (3D→2D bridge)
Step 6: DockSpace 布局 + 多窗口管理
```

## 技术栈总结

```
渲染引擎层 (已有)
  ├─ D3D11 独立设备 / SwapChain / 管线
  ├─ Vertex Shader + Pixel Shader 运行时编译
  └─ 顶点缓冲 / 索引缓冲 / 常量缓冲

UI 引擎层 (已有)
  ├─ ImGui (Win32 + DX11 Backend)
  ├─ Windows 窗口 (WS_EX_LAYERED / TOPMOST)
  └─ Raw Input 键盘

验证中
  ├─ ImPlot (图表库，可选)
  ├─ stb_image (纹理加载)
  ├─ Render-to-Texture (D3D11 渲染到纹理)
  └─ 游戏世界 → 屏幕坐标投影
```
