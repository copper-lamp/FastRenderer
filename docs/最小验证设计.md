# 文件热加载 GUI 渲染引擎 — 最小验证设计

## 验证目标

实现 JSON 声明式 GUI 定义 → ImGui 渲染引擎 + 文件热加载的核心链路。

## 待验证的能力

1. **JSON 解析器** — 将 JSON GUI 定义树解析为内存中的控件节点树
2. **ImGui 渲染器** — 将控件节点树映射为 ImGui 控件调用
3. **文件热加载** — 监控 JSON 文件变更，自动刷新 UI
4. **canvas 画布** — 自定义绘制指令序列支持

## 支持的组件（最小集）

| 类型 | 对应 ImGui 控件 | 优先级 |
|------|----------------|--------|
| `window` | Begin/End | P0 |
| `column` | 垂直布局 | P0 |
| `row` | 水平 SameLine | P0 |
| `button` | Button | P0 |
| `text` | Text | P0 |
| `checkbox` | Checkbox | P0 |
| `slider` | SliderFloat | P0 |
| `separator` | Separator | P0 |
| `input` | InputText | P1 |
| `combo` | Combo | P1 |
| `canvas` | 自定义绘制指令 | P1 |
| `group` | BeginGroup/EndGroup | P1 |

## 数据结构

```cpp
// 控件节点
struct GuiNode {
    std::string type;          // 控件类型
    std::map<std::string, std::string> props;  // 属性键值对
    std::vector<GuiNode> children;  // 子节点
};

// 完整 GUI 定义
struct GuiDefinition {
    GuiNode root;              // 根节点
    std::string id;            // 唯一标识
    std::string sourceFile;    // 来源文件路径
};
```

## 文件热加载机制

```
JSON 文件目录 (gui_defs/)
    │
    ├── FileWatcher 线程
    │   ├── std::filesystem::last_write_time 轮询
    │   ├── 检测到变更 → 标记为脏
    │   └── 下一帧重新解析 + 渲染
    │
    ├── 示例 JSON 文件
    │   └── example.json
    │
    └── 热加载目录
```

## 集成方式

作为验证菜单的第 8 个模块（`activeVerificationModule = 8`），菜单中显示：
- 当前加载的 JSON 文件列表
- 文件变更实时显示
- 基础调试信息

## 实施计划

4. 创建 `gui_defs/` 目录 + 示例 JSON
